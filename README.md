## Project Notebook

The projects below are described through the skills and ideas I practiced while building them.

<details open>
<summary><b>XR Simulation Interface</b></summary>

This project area explores how simulation data can become easier to understand inside an immersive environment. I use Unity and XR ideas to represent technical systems visually, add runtime interaction, and experiment with spatial control panels.

```text
Learning signal : Unity + XR + simulation + runtime visualization
What it taught  : scene structure, networking basics, runtime loading, spatial UI, debugging on headset
Build focus     : making technical simulation data feel more visual, interactive, and understandable
```

</details>

<details>
<summary><b>Behavior Logic And Feedback</b></summary>

This work is about making systems decide actions step by step. I use behavior-tree thinking to organize control flow, connect it with simulation, and observe how feedback can guide a system toward a target state.

```text
Learning signal : behavior trees + simulation + monitoring + feedback loops
What it taught  : control logic, state observation, debugging, verification, modular actions
Build focus     : turning a complex control problem into smaller readable decision steps
```

</details>

<details>
<summary><b>Device And Network Utility</b></summary>

This project area focuses on managing small devices from a laptop. It includes ideas like discovering devices on a local network, showing their state in a dashboard, sending commands carefully, and adding rollback-style safety when a change could disconnect a device.

```text
Learning signal : Python + local networking + dashboard UI + safety checks
What it taught  : device discovery, command validation, browser UI, authentication concepts, recovery design
Build focus     : making repeated device-management tasks easier to see, control, and recover from
```

</details>

<details>
<summary><b>Mixed Reality Prototypes</b></summary>

These are Unity learning builds where I test how virtual objects can live inside a physical room. The experiments include surface targeting, controller input, placing points in space, previewing paths, and using debug panels to understand what is happening inside the headset.

```text
Learning signal : XR interaction + passthrough concepts + room-aware placement
What it taught  : input routing, spatial anchors, visual feedback, headset debugging, iterative prototyping
Build focus     : testing how virtual objects can respond naturally to physical-room context
```

</details>

<details>
<summary><b>Digital Twin Experiments</b></summary>

I use virtual models to understand how software can represent real movement. One learning direction is connecting a Unity-side visual twin with hardware-style commands so that the virtual motion helps explain what the physical system is doing.

```text
Learning signal : Unity animation + microcontroller communication + hardware behavior
What it taught  : HTTP requests, command acknowledgement, motion mapping, virtual feedback
Build focus     : connecting physical motion ideas with a readable virtual representation
```

</details>

<details>
<summary><b>Public Mini Projects</b></summary>

Small public projects help me practice fundamentals. These include experiments around alternate ways of displaying information, embedded-style logic, and basic web project structure.

```text
Learning signal : small scope + clear logic + readable documentation
What it taught  : timing, UI basics, repository hygiene, incremental learning
Build focus     : practicing fundamentals through compact, understandable projects
```

</details>
