## Featured Learning Projects

### XR Simulation Interface

> Immersive prototype work for understanding technical systems through Unity and XR.

- Visualizes simulation ideas inside an interactive 3D environment.
- Practices runtime visualization, spatial UI, headset debugging, and interaction flow.
- Helps connect robotics-style data with a more intuitive visual experience.

### Behavior Logic And Feedback System

> Learning setup focused on structured decision-making and feedback-driven control.

- Uses behavior-tree thinking to break complex actions into readable steps.
- Practices monitoring, state observation, simulation feedback, and verification.
- Helps build intuition for how robotic systems can sense, decide, and act.

### Device And Network Utility

> Python-based workflow for managing small connected devices from a laptop.

- Practices local discovery, dashboard-style UI, command validation, and recovery flow.
- Focuses on making repeated device-management tasks easier to understand and operate.
- Strengthens networking, Python scripting, and practical tool-building fundamentals.

### Mixed Reality Interaction Prototype

> Unity XR experiments around physical-room interaction and spatial placement.

- Tests surface targeting, controller input, point placement, path preview, and debug panels.
- Helps understand how virtual objects can behave naturally in real surroundings.
- Builds confidence with headset-first development and iterative debugging.

### Digital Twin Experiment

> A Unity and hardware-oriented learning project for representing real movement virtually.

- Connects visual motion with hardware-command concepts.
- Practices request handling, motion mapping, command acknowledgement, and virtual feedback.
- Explores how a digital model can make physical behavior easier to reason about.

### Public Mini Builds

> Smaller projects for practicing fundamentals and keeping the learning loop active.

- Embedded-style logic and alternate display ideas.
- Web basics using HTML, CSS, and JavaScript.
- Clear documentation, small scope, and steady improvement.
